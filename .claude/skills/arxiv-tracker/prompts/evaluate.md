## 评估维度

对每篇论文，评估以下四个维度：

1. **前瞻性 (novelty, 1-5)**
   - 5: 开创性方向或重大突破，提出全新范式
   - 4: 显著创新，对领域有推动作用
   - 3: 有价值的增量贡献，扎实的改进
   - 2: 常规跟进，贡献有限
   - 1: 跟随性工作或已过时方向

2. **方法严谨性 (rigor, 1-5)**
   - 5: 完善的实验设计、充分的 baseline 对比、消融实验、统计显著性检验
   - 4: 实验设计合理、有对比但可能缺少部分分析
   - 3: 标准实验流程，结果可信但缺少深入分析
   - 2: 实验不够充分，对比不足
   - 1: 实验设计存疑，结论缺乏支撑

3. **相关性 (relevance, 1-5)**
   - 5: 直接命中知乎数据业务核心需求（评测/数据合成/Agent），有明确产业落地价值
   - 4: 高度相关，方法或结论可直接借鉴到业务中
   - 3: 间接相关，需泛化或适配才能结合
   - 2: 边缘相关，与业务需求脱节
   - 1: 仅表面关键词匹配，实质不相关

4. **合作潜力 (collaboration, 1-5)**
   - 5: 直接可落地为知乎专家数据项目，方向精准且社区有现成专家池
   - 4: 方向匹配，需要一定组织成本但可行
   - 3: 有间接关联，需要泛化或适配才能结合
   - 2: 方向偏学术，与产业实践脱节
   - 1: 纯基础研究或与知乎资源完全不匹配

## 评分规则

- 纯基础模型研究（架构/优化器/预训练细节）：除非揭示了"数据→模型能力"的关系，否则 relevance 和 collaboration 应给低分
- 中国模型厂商论文（见上方列表）：如果是模型发布或技术报告，relevance 应给较高权重
- 仅从摘要判断方法严谨性时要保守，给出合理估计而非猜测

## 兴趣分类 (interest_type)

- `business`: 与知乎数据业务相关（能力评测 + 产业落地 + 数据链路 + 客户动态）→ 进入主报告
- `personal`: 有学术/研究价值但非业务优先（纯基础研究、架构探索、理论分析、个人兴趣方向如模型行为设计/对齐理论）→ 进入"你可能感兴趣的"

## 标签 (tags)

从以下预定义集合中为每篇论文选择 1-4 个最相关的标签：

- `能力评测`: benchmark 设计、评测方法、模型对比
- `Agent`: 智能体架构、多智能体、工具调用
- `数据合成`: 训练数据生成、数据质量、蒸馏
- `前沿发布`: 头部实验室模型发布/技术报告
- `多模态`: 视觉-语言、音频、跨模态
- `沙盒评测`: 博弈环境、交互式评测、模拟器
- `RL/对齐`: 强化学习、RLHF、偏好对齐
- `安全`: 安全评测、对齐、红队
- `产业应用`: 金融/医疗/法律等垂直领域实际场景
- `社区相关`: 与知乎社区属性/专家组织有关联
- `重点关注客户`: 中国模型厂商相关论文

## 输出格式

严格输出 JSON 数组，不要包含其他文字：

```json
[
  {
    "arxiv_id": "2403.xxxxx",
    "novelty": 4,
    "rigor": 3,
    "relevance": 5,
    "collaboration": 4,
    "overall": 4.1,
    "interest_type": "business",
    "tags": ["能力评测", "Agent", "产业应用"],
    "affiliations": "Harbin Institute of Technology, Alibaba Group",
    "reason": "一句话说明为什么值得/不值得关注",
    "highlight": "核心技术贡献的一句话总结",
    "action_hint": "可在xx场景下重点关注xx方法/思路"
  }
]
```

计算公式：overall = novelty × 0.25 + rigor × 0.15 + relevance × 0.35 + collaboration × 0.25

注意：
- 每篇论文都必须评估，不要遗漏
- reason 要具体，不要泛泛而谈
- action_hint 给出实操建议：这篇论文的方法/思路可以在什么场景下怎么用
- tags 至少 1 个，最多 4 个
- affiliations 是**必填字段**，不要省略。从摘要文本和作者名推断主要作者的机构/大学/公司，用逗号分隔。如果摘要中有明确提及则使用，否则基于作者名和已知信息推断（如知名团队/实验室/大厂）。实在无法判断时填 "Unknown"
